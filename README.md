# RS Meets: Find the Bug 🐞

**RS School Community Event**

Welcome to the bug-hunting challenge! This repository contains 5 small web pages, each with **5 intentional CSS bugs**. Your task is to find and fix all the bugs.

Difficulty increases from Task 1 (easiest) to Task 5 (hardest).

## 🔗 Deploy

> **Live demo:** https://oreskag.github.io/RS-Meets-html-css-bugfixing/index.html

---

## How to start

1. Fork this repository
2. Clone it to your computer
3. Open `index.html` in a browser — this is the main landing page
4. Navigate to each task using the links in the header
5. Compare what you **see** with what is **described** below
6. Find the bugs in the CSS files and fix them
7. Commit your changes and push

> **Hint:** All bugs are in the `styles.css` files. The HTML is correct — don't change it!

## Task 1 — My Pet ⭐

A simple pet profile card. Something looks off...

| # | What's wrong |
|---|---|
| 1 | The page title "My Lovely Pet" is invisible — it should be dark text on a light background |
| 2 | All content inside the card is aligned to the right — it should be centered |
| 3 | The pet image is not visible — it should appear as a 300px-wide photo |
| 4 | The pet's name "Buddy" has a strikethrough — it should be normal text without any line |
| 5 | The "Adopt Me" button is gray — it should be green (#4CAF50) |

---

## Task 2 — Cafe Menu ⭐⭐

A cozy cafe menu page. But the design is broken...

| # | What's wrong |
|---|---|
| 1 | The page background is pitch black — it should be a light cream color (#fff8f0) |
| 2 | The cafe name "Cozy Beans Cafe" has disappeared — it should be visible on the page |
| 3 | Menu section borders are thick red dashes — they should be a thin solid light border |
| 4 | The numbered list `<ol>` has no numbers — items should be numbered 1, 2, 3... |
| 5 | Menu items are squished together — each item should have vertical padding for spacing |

---

## Task 3 — Travel Blog ⭐⭐⭐

A travel blog post about a mountain trip. But it's hard to read...

| # | What's wrong |
|---|---|
| 1 | The article is way too narrow — it should be about 800px wide |
| 2 | The date "March 15, 2026" is invisible — it should be visible as gray text |
| 3 | The mountain photo is almost transparent — it should be fully visible (100% opacity) |
| 4 | Text lines overlap each other — line spacing should be comfortable for reading |
| 5 | The entire card has become a circle — it should have normal slightly rounded corners |

---

## Task 4 — Photo Gallery ⭐⭐⭐⭐

A photo gallery should show images in a nice grid. This one... doesn't.

| # | What's wrong |
|---|---|
| 1 | All photos are in a single column — they should be displayed in a flex grid (3 per row) |
| 2 | Photos don't wrap to the next row — cards should wrap when they don't fit |
| 3 | There's no space between the photo cards — there should be a ~20px gap |
| 4 | Gallery is aligned to the right side — it should be centered |
| 5 | Photos look stretched and distorted — should be rounded in cards. |

---

## Task 5 — Portfolio ⭐⭐⭐⭐⭐

A portfolio page with project cards. The hover effects and layout need fixing.

| # | What's wrong |
|---|---|
| 1 | Project cards don't wrap on smaller screens — they should wrap to the next line |
| 2 | Cards don't react to hover — they should lift up and show a stronger shadow on hover |
| 3 | Hovering over a card has no smooth transition — the change should be animated |
| 4 | Tags (HTML, CSS, etc.) are stacked vertically — they should be in a horizontal row |
| 5 | "View Project" link has a default arrow cursor — it should show a pointer (hand) cursor |
